건강 one app ERD 보드

발표용 요약 ERD입니다. 핵심은 모든 건강 기록을 health_events에 모으고, 세부 정보만 도메인 테이블로 나누는 구조입니다.

파랑: 사용자/출처 초록: 중심 이벤트 회색: 생활 기록 도메인 노랑: 사진/AI 분석 빨강: 복용/주의정보

사용자와 입력 출처

users 회원 계정, 이메일, 이름
user_profiles 키, 몸무게, 알레르기, 시간대
data_sources manual, vision_ai, label_scan, apple_health, samsung_health

사진과 AI 분석

media_assets 식사 사진, 음료 라벨, 화장품 성분표
ai_analysis_results 모델명, 신뢰도, 분석 결과, 안전 문구
Core Timeline

health_events

활동, 수면, 식사, 음용, 복용, 화장품 사용을 같은 시간축에 저장합니다.

사용자/출처 -> health_events -> 도메인 상세 기록
외부 Health API 데이터는 source_id + external_record_id로 중복 생성을 막습니다.

생활 기록 도메인

activity_records 걸음, 운동 시간, 활동 칼로리
sleep_records 수면 시간, 수면 단계, 수면 점수
meal_logs + food_items 식사, 음식 항목, 칼로리, 영양소
beverage_logs 수분, 커피, 캔음료, 카페인, 당류
cosmetic_products + ingredients 화장품 제품, 성분, 사용 반응

복용과 알림

medication_items + supplement_items 약/영양제 품목
intake_logs 실제 복용 기록
interaction_checks 상호작용 주의정보, 전문가 상담 권고
reminder_rules + notification_logs 복용/수분/생활 알림

왜 중심 테이블이 필요한가

모든 기록을 같은 시간축에 놓아야 일간/주간 리포트에서 수면, 활동, 식사, 복용, 화장품 사용을 한 번에 합칠 수 있습니다.

사진 분석 구조

사진 원본은 media_assets, AI 결과는 ai_analysis_results에 저장해서 식사/음료/화장품 분석이 같은 구조를 씁니다.

안전선

약/영양제 분석은 진단이 아니라 건강 관리용 주의정보입니다. 고위험 또는 불확실한 조합은 전문가 상담 권고를 포함합니다.
