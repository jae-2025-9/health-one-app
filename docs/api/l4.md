# L4: 알림 · 리포트 · 연동 API 명세

담당: L4 (D — 프론트)  
테이블: `reminder_rules`, `notification_logs`  
엔드포인트: `/reminders` (GET·POST), `/reminders/{id}` (PATCH), `/reports/daily` (GET), `/reports/weekly` (GET), `/integrations/health-sync` (POST)

> **통합 주의사항**  
> - `/reports/*` 응답은 L1~L3 도메인 필드명 동결 후 최종 확정. 여기 정의한 필드명은 현재 schema.sql 기준이며 통합 시 조정 가능.  
> - `/integrations/health-sync` dedup 키는 `sourceId + externalRecordId` → `health_events_source_external_uidx`와 일치.  
> - 응답 봉투: `{ data, meta }` / 오류: `{ error, meta }` (L1 동결 사양 그대로).

### POST /integrations/health-sync — 외부 건강 데이터 동기화

**응답 타입 변경 요청**: `ResourceResponse` → `HealthSyncResponse` (data: HealthSyncResult)

**요청 바디**: `HealthSyncRequest` (기존 스키마)

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `sourceType` | string (enum) | ✓ | `apple_health` \| `samsung_health` |
| `externalAccountId` | string \| null | | 외부 계정 식별자 (있으면 data_sources 매핑에 사용) |
| `events` | HealthEventCreate[] | ✓ | 업로드할 이벤트 배열 |

각 이벤트의 `externalRecordId`는 외부 플랫폼의 고유 ID.

**dedup 규칙**: `sourceId + externalRecordId` 조합이 이미 존재하면 삽입하지 않고 `deduplicated` 카운트 증가 (`health_events_source_external_uidx` 인덱스 사용).

**요청 예시**

```json
{
  "sourceType": "apple_health",
  "externalAccountId": "user-apple-id-abc123",
  "events": [
    {
      "eventType": "activity",
      "sourceType": "apple_health",
      "externalRecordId": "AH-STEP-20260603-001",
      "startedAt": "2026-06-03T00:00:00+09:00",
      "endedAt": "2026-06-03T23:59:59+09:00",
      "timezone": "Asia/Seoul",
      "confidenceScore": 1.0,
      "rawPayload": { "steps": 8432, "activeMinutes": 47 }
    },
    {
      "eventType": "sleep",
      "sourceType": "apple_health",
      "externalRecordId": "AH-SLEEP-20260602-001",
      "startedAt": "2026-06-02T23:00:00+09:00",
      "endedAt": "2026-06-03T07:00:00+09:00",
      "timezone": "Asia/Seoul",
      "confidenceScore": 0.95,
      "rawPayload": { "totalMinutes": 420 }
    }
  ]
}
```

**성공 응답 202** (비동기 처리 — 요청 수락)

```json
{
  "data": {
    "sourceId": "c3d4e5f6-0000-0000-0000-000000000003",
    "sourceType": "apple_health",
    "received": 2,
    "inserted": 1,
    "deduplicated": 1,
    "failed": 0
  },
  "meta": {
    "requestId": "req_20260603_0006",
    "generatedAt": "2026-06-03T09:05:00+09:00"
  }
}
```

**서버 처리 흐름**

1. `sourceType` + `externalAccountId`로 `data_sources` 조회 → 없으면 새로 생성 → `sourceId` 확보.
2. 각 이벤트마다 `(sourceId, externalRecordId)` 조합으로 `health_events_source_external_uidx` 확인.
3. 이미 존재하면 `deduplicated++`, 없으면 INSERT → `inserted++`.
4. 제약 위반 (타입 오류 등)이면 해당 이벤트 `failed++` 후 계속 처리.

**오류**

| 코드 | HTTP | 조건 |
|---|---|---|
| `VALIDATION_ERROR` | 400 | `events` 배열 비어있음, `sourceType` 미허용 값 |
| `UNAUTHORIZED` | 401 | JWT 없거나 만료 |
